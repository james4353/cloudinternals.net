How to host a WordPress website on AWS using Amazon Lightsail

AWS offers multiple ways of running applications. One of the most versatile services is EC2 which is where you can create virtual machines. The thing about EC2 is they may be overwhelming for simple use cases. For common use cases such as creating a basic virtual machine and starting a WordPress website you might want to have a simpler alternative. Amazon Lightsail is intented to simplify achieving such basic goals.

In this post, we are going to look into creating a new WordPress website with Amazon Lightsail.

What is Amazon Lightsail?

AWS services are generally like LEGO bricks. You can mix and match them to build complex infrastructure. This comes at a cost though. Sometimes all you need is a pre-packaged simple solution to achieve a simple task. There are some hosting providers out there that offer virtual private servers (VPS) with one-click application deployments such as Linode and Digital Ocean. Amazon Lightsail is AWS‚Äôs solution offered to this market.

It was announced in 2016. Unlike an EC2 instance, you don‚Äôt select individual components of a server. You rather select a pricing plan which includes a pre-packaged server with a configuration proportional to the pricing. This alone simplifies a lot of tasks that might be daunting to people who are not tech-savvy and just want to get a site up and running as fast as they possibly can.

Without further ado, let‚Äôs take a look how Lightsail works.

Getting Started

When you visit Amazon Lightsail Dashboard first thing you notice is that it‚Äôs quite different than a regular AWS service:

It might take some time getting used to the this UI but fortunately it‚Äôs quite intuitive so getting everything set up should not be too much of a hassle.

Let‚Äôs get started with our WordPress site:

  1. In the dashboard, Click Create Instance button in the middle of the screen.

  1. First thing you need to do is to approve or change the Instance location. The closest region to your location is pre-selected.

When choosing the instance location, be mindful about your target audience. Being close to your users will reduce the network latency and will make a better experience for your users.

  1. Next, you pick your operating system image.

In terms of both price and performance, I‚Äôd strongly recommend choosing Linux/Unix as your platform which is what we‚Äôll be selecting in this article.

Pricing

In terms of pricing, there is no extra charge for using function URLs. The cost of Lambda executions is calculated the same way whether or not they were invoked from a browser over the public Internet or a CLI inside your company network. The details of pricing can be found here

The duration is calculated as per the below rule:

Duration is calculated from the time your code begins executing until it returns or otherwise terminates, rounded up to the nearest 1 ms*

This means if you implement authentication and receive unauthenticated calls, you don‚Äôt pay for those calls, which might be helpful to be charged for DDoS attacks.

Lambda Function URLs vs Amazon API Gateway

In the introduction section, I mentioned that we used Amazon API Gateway for public APIs before the function URLs feature. So even though the feature helps a lot for specific use cases, I think it‚Äôs worth comparing both to have more realistic expectations.

Amazon API Gateway features

  • Authentication and authorization
  • Request throttling
  • Usage plans and quotas
  • API Keys
  • AWS WAF integration
  • Web socket support
  • CORS support
  • Built-in custom domain support

Function URLs

  • AWS_IAM authentication
  • CORS support

So we have lots of missing stuff in function URLs. The question is, do we need all of them for all our APIs? The chances are that a single public endpoint is good enough for many use-cases, and it costs a lot less.

Conclusion

I needed a simple API to carry out a simple task on multiple occasions. Now that we have the ability to develop APIs that can be invoked from anywhere is a significant improvement on the server overall.
